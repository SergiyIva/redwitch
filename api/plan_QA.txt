## Запуск/Перезапуск/Остановка
* В каталоге с проектом в терминале выполнить:
    > forever start ./src/index.js
* Для перезапуска:
    > forever restart ./src/index.js
* Для остановки:
    > forever stop ./src/index.js

## Общее
* Весь ресурс должен корректно отображаться на любом устройстве, при возникновении
проблем с адаптацией для конкретного устройства - уведомить с предоставлением
информации об устройстве (модель, разрешение экрана, ОС, браузер)
* При возникновении необходимости в увеличении функциональности - сформулировать
потребность в следующем виде:
    1. Основное назначение.
    2. Требуемая функциональность.
    3. Примеры реализации (если есть).
    4. Вариант(ы) представления. (Как по Вашему это должно быть встроено в ресурс,
    на одной из существующих страниц и какой / на новой и по какому url (адресу),
    возможно есть пожелания о внешнем виде нового элемента и т.д.)

## Администрирование
* По адресу /admin/add-card можно добавить карточку на главной странице.
* Ссылки на контакты нужно задать отдельно, либо изменить на другие.
* Любые графические представления, текстовая инфа, поля форм, порядок следования
элементов, цвета палитры м.б. заменены.
* Для автоматизации отправки электронной почты необходимо выбрать ретранслятор SMTP.
* Если потребуется автоматизация получения сервером и последующей обработки электронной
почты, то стоит разобраться с IMAP (imap-simple).
* Важные логи операций сервера пишутся в файл start.log, a логи запросов в access.log

## Функциональность
* Проверки на корректность номера телефона, имени, емаил.
* Каждый пользователь, посетивший ресурс, получает уникальный ID, в последствии
это дает возможность контроля посещаемых им страниц, сбора статистики для дальнейшей
аналитики собранных данных, а также превентивных мер по предупреждению некоторых видов
атак на ресурс. Уникальный Id хранится на клиенте 24 часа, затем автоматически удаляется.
* Максимальное количество заявок, которое может поступить с 1-го устройства
за сутки - 11шт. Настраиваемо (все дальнейшее также опционально, при
желании можно изменить, данная инфа больше указываться не будет для краткости).
* При указании не существующего пути будет выполнен переход с кодом 404.
* При ошибке на стороне сервера отобразится код 500.
* При ошибке на стороне сервера во время оставления заявки пользователем, будет
отображено уведомление об ошибке.
* При успешном окончании операции по заявке, будет отображено уведомление об успехе операции.

## Безопасность
* Из соображений безопасности в заголовках, получаемых клиентом, указывается, что
разработка серверной части была реализована на PHP, к реальности это не имеет никакого
отношения и сделано только для предотвращения целевых атак злоумышленников. На
работу сервера, клиента и производительность это никак не влияет.
* Соединение установливается как по HTTPS, так и по HTTP, если нужно, можно включить автоматическую
переадресацию на безопасное соединение. На данный момент такая реализация только в административной
части приложения.
